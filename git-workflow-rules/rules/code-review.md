# Code Review Rules

## レビュイー（PR 作成者）

PR 提出前にセルフレビューを行うこと:

- 重複コードの排除
- 命名の一貫性（変数・関数・ファイル）
- 不要なコード・コメントの削除
- 関数・クラスの責務が単一になっているか
- 入力値のバリデーション漏れがないか
- 機密情報（APIキー・トークン等）のハードコーディングがないか

## レビュアー

以下の観点でレビューを行うこと:

- **機能:** 要件を満たしているか
- **可読性:** 命名・責務分離が適切か
- **セキュリティ:** workflow のセキュリティレビューと同観点
- **テスト:** カバレッジは十分か

## コメントの書き方

- `nit:` — 細かい提案（マージをブロックしない）
- `MUST:` — 必須修正（マージ前に対応が必要）
